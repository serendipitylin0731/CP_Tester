# CP Tester（奶娃版）

一款专为算法竞赛和编程练习设计的 VS Code 插件，提供便捷的本地测试与评测功能。

## 功能简介

CP Tester 在 VS Code 侧边栏提供一个测试面板，帮助你在提交代码前快速验证程序正确性。支持手动输入测试数据或从文件夹批量加载，自动编译运行并比对输出结果。

### 核心特性

- **多语言支持**：自动识别并编译运行 `.c`、`.cpp`、`.py` 文件
- **灵活的数据输入**：可手动添加最多 5 组测试数据，或从文件夹批量导入 `.in` / `.out`（`.ans`）数据对
- **完整的评测结果**：显示每个测试点的状态、运行时间和内存占用
- **可自定义限制**：支持调整时间限制（默认 1000 ms，最大 60000 ms）和内存限制（默认 256 MiB，最大 2048 MiB）
- **状态图片展示**：根据整体评测结果在面板顶部显示对应的动态图片

### 评测状态

| 状态 | 说明 |
|------|------|
| `Accepted` (AC) | 答案正确 |
| `Wrong Answer` (WA) | 输出与期望不匹配 |
| `Time Limit Exceeded` (TLE) | 运行时间超过限制 |
| `Memory Limit Exceeded` (MLE) | 内存使用超过限制 |
| `Runtime Error` (RE) | 程序运行时出错 |
| `Compile Error` (CE) | 编译失败 |
| `Pending` | 正在评测中 |

## 使用方法

### 打开面板

点击 VS Code 左侧活动栏的 **烧杯图标**（Beaker）打开 CP Tester 面板，或拖动到 `KIMI CODE` 图标右侧来回切换使用。

### 手动输入测试

1. 在编辑器中打开需要测试的程序文件（`.c` / `.cpp` / `.py`）。
2. 在 CP Tester 面板中点击 `+` 按钮添加测试用例。
3. 在输入框中填写**输入数据**和**期望输出**。
4. （可选）在配置区域修改时间和内存限制。
5. 点击 `▶ Run` 按钮开始评测。
6. 查看下方结果区域，每个测试点会显示状态、时间和内存信息。

### 从文件夹加载数据

1. 准备一个文件夹，其中包含成对的测试数据文件：
   - 输入文件：`.in` 后缀
   - 输出文件：同名的 `.out` 或 `.ans` 后缀
2. 在面板中点击 `📁` 按钮，选择该文件夹。
3. 插件会自动加载所有合法的数据对并显示在面板中。
4. 点击 `▶ Run` 即可批量评测。

### 图片展示

面板顶部会根据当前评测结果展示对应的图片，支持以下状态文件夹：

- `sets/None/` — 未开始
- `sets/Pending/` — 评测中
- `sets/AC/` — 全部通过
- `sets/CE/` — 编译错误
- `sets/RE/` — 运行时错误
- `sets/MLE/` — 内存超限
- `sets/TLE/` — 超时
- `sets/WA/` — 答案错误

将 `.png` 或 `.gif` 图片放入 `media/sets/` 下对应的状态文件夹中，插件会随机抽取展示。

## 配置项

在 VS Code 设置中搜索 `CP Tester` 可修改以下选项：

| 配置项 | 默认值 | 范围 | 说明 |
|--------|--------|------|------|
| `cpTester.timeLimit` | 1000 | 1 ~ 60000 | 默认时间限制（毫秒） |
| `cpTester.memoryLimit` | 256 | 1 ~ 2048 | 默认内存限制（MiB） |

## 注意事项

- 运行测试前请确保已保存当前文件。
- 时间/空间超限判断：超过设定值即判定为超限；超过设定值两倍时会被强制截停。
- 输出比对时忽略行尾空格差异。
